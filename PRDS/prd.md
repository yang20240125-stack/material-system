# 📦 AI 驱动的企业物资管理系统 PRD

## 1. 产品概述
本项目是一个基于 Web 的企业物资管理系统。系统旨在通过数字化手段管理公司内部物资的入库、领取、归还及库存状态。核心亮点在于集成了一个全局悬浮的 AI 助手，用户及管理员可以通过自然语言对话直接进行数据查询与物资的增删改查（CRUD）操作。同时，系统深度接入钉钉生态，实现无缝登录与消息流转。

## 2. 目标角色与权限控制 (RBAC)
系统采用基于角色的访问控制，核心区分以下两类身份，且权限需在前端 UI、后端 API 及 AI 助手三个层面严格校验：

* **普通用户 (User)**：
    * 仅可浏览非特殊的普通物资。
    * 可提交普通物资的领取申请。
    * 可与 AI 助手对话查询物资库存及个人领用记录。
* **系统管理员 (Admin)**：
    * 拥有系统最高权限。
    * 可对所有物资进行增、删、改、查（通过系统界面或 AI 助手）。
    * 专属权限：可直接操作、领取或分配“特殊/贵重物资”。
    * 可查看全公司的物资流转台账（领取人、时间、状态）。
    * 可进入后台系统配置中心。

## 3. 核心功能模块

### 3.1 钉钉深度接入模块 (Auth & Notification)
* **企业内免密登录 (SSO)**：
    * 作为钉钉企业内部 H5 微应用接入。
    * 前端调用钉钉 JSAPI 获取免登授权码 (`authCode`)。
    * 后端通过 `authCode` 调用钉钉接口换取用户的钉钉 `userid`。
    * 系统根据 `userid` 匹配本地数据库，完成静默登录或自动注册（同步姓名与头像）。
* **消息推送功能 (钉钉工作通知)**：
    * **领用通知**：用户成功领取物资后，系统通过钉钉 API 向该用户发送“物资领取成功”的卡片消息（包含物资名称、数量、时间）。
    * **分配通知**：管理员将“特殊物资”分配给指定用户时，该用户会收到专门的钉钉消息提醒。

### 3.2 物资管理与系统配置模块
* **物资大厅展示**：卡片或表格列表展示物资（图片、名称、分类、库存数量、状态、物资级别）。
* **图片管理**：支持物资图片的配置（输入图片 URL 或支持基础的本地上传至对象存储）。
* **检索与筛选**：提供按关键词、分类、库存状态的高效检索。
* **后台配置中心 (Admin 专属)**：
    * **AI 模型配置**：提供 UI 表单供管理员动态配置大模型的 `Base URL` 和 `API Key`。
    * **钉钉配置**：提供 UI 表单配置钉钉的 `AppKey` 和 `AppSecret`。
    * *注：以上敏感配置项在数据库中需加密存储，且前端仅作掩码展示。*

### 3.3 悬浮 AI 助手 (核心交互亮点)
* **全局悬浮气泡**：网页右下角常驻，点击展开对话窗口。
* **上下文与权限感知**：AI 需自动读取当前登录用户的会话 (Session) 角色。若是普通用户，AI 拒绝执行修改或删除数据库的指令。
* **Function Calling (工具调用)**：
    * *查询意图*：对接数据库查询接口（如：“查一下公司还有多少个鼠标？”）。
    * *操作意图 (CRUD)*：对接数据的增删改接口（如：“把A4纸的库存加上 50 箱”）。
* **操作确认与数据对比 (Diff Preview)**：
    * **强规则**：当 AI 识别到“修改”或“删除”等破坏性/变更性操作时，**严禁直接操作数据库**。
    * **交互流**：AI 必须在聊天框内返回一个卡片，展示**修改前的数据信息**和**修改后的数据信息**。
    * 用户在卡片上点击“确认执行”后，前端才发起最终的写操作请求。

### 3.4 领取与流转记录模块 (台账)
* **物资领取**：通过系统界面按钮或 AI 对话发起物资领取动作。
* **特殊物品拦截**：普通用户尝试领取标记为“特殊 (SPECIAL)”的物资时，系统拦截并提示“仅限管理员分配”。
* **流转日志**：任何人、任何形式的库存变动，都必须生成不可篡改的日志记录，供管理员追溯查询。
